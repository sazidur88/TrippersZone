1.0 Introduction

1.1 PURPOSE OF THIS DOCUMENT

The purpose of this design document is to present the system design plan of the product named "TrippersZone" by which anyone can discover the nearest places for accommodation. It's one kind of paying guest accommodation system. Besides home searching in our application, users can search other services like : restaurants, hospitals, police stations etc. House owners can give rent their houses and can get some profit. It will also solve hotel room booking crisis within a reasonable price.This Design Document will check that the present outline meets the majority of the express prerequisites contained in the genuine model and also the important necessities wanted by the user. It likewise intends to give a more point by point comprehension of the outline and design of the "TrippersZone". Topics covered include the following:

  • Detailed description of components
  • User interface design
  • Design component architecture
  • Processing narratives
  • Design constraints and restrictions
  • Test cases and expected results

1.2 SCOPE OF PROJECT

We observed some issues in our country, when there are festivals all of the people like to go in well recognized places and then the problems start like, hotel management charges 1o times more money than a normal days. Even in some cases they can not find the room due to overcrowd. Our application can solve this problem. A random house can be cost lower than hotels. It can reduce the pressure of overloads.

1.3 DEFINITIONS, ACRONYMS AND ABBREVIATIONS

  • SDS – Software Design Specification. This document represents the requirements that the software must fulfill.
  • User – Any person who uses the software.
  • Administrator/Admin– A person who has administrative privilege/access to the system .
  • Home page- The 1st page that user will see while visiting the application.
  • Registration Page– The initial page to allow users to register.
  • Login Page– The initial page to allow registered user to login .
  • Profile- The personal page with all information for each users who have logged in.
